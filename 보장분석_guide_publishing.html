Publishing Guide

0. 개요

0.1. 서버 경로 및 구조

  • /cg.sag.onweb/src/main/webapp/resources/
  • └ css / *.css
  • └ html / *.html
  • └ images

    └ common / *.* *가입설계,영업지원 이미지

    └ contents / *.* *가입설계,영업지원 이미지

    └ . . .

  • └ js / *.js
  • ※ 파란색으로 표기된 폴더명, 위치는 변경할 수 없다.

1. HTML

1.1. 위치 및 정의

  • ① /html/보장분석_*.html
    개발화면을 위한 html Markup File
  • ② /html/보장분석_guide_**.html
    퍼블리싱 가이드(html Markup Guide).
    유지 보수를 위해 스타일 정의 및 패턴 등 정보제공 문서
  • ※ html 파일은 개발, 운영에 업로드 하지 않는다.

1.2. 파일명 정의

  • 화면정의서의 화면ID 또는 메뉴, 화면명의 조합으로 명명한다.
  • 가입설계, 영업지원 파일과의 구분을 위해,
    파일명 앞에 '보장분석_'을 붙여준다.
    ex)보장분석_main.html, 보장분석_guide_button.html

2. CSS

2.1. 위치 및 정의

  • /css/*.css
  • ① UI Design
    • gu.common.css
      reset, common style, form, button 등 기본(공통) 스타일 정의
    • gu.layout.css
      페이지, 팝업 등 레이아웃 스타일 정의
    • gu.contents.css
      컨텐츠 영역 스타일 정의

    ※ 현재 운영 파일
    common.gu.css, layout.gu.css, contents.gu.css

  • ② ngDialog UI
    • ngDialog.min.css
    • ngDialog-theme-default.min.css
    • gu.ngDialog-theme-meritz.css
      보장분석의 UI에 맞게 스타일 재정의

    ※ library, plug-in 에서 제공하는 원본 파일을 직접 수정하지 않음. 별도의 파일을 지정하여 스타일 보완.

    ※ 현재 운영 파일 ngDialog-theme-meritz.gu.css 이며,
    동일한 파일에 스타일 추가하여 사용할지, 새파일을 만들어 정의할지는 편한대로 선택

  • ③ Desktop, Mobile 구분
    모바일 문서는 mb를 붙어준다.
    ex) gu.mb.common.css, gu.mb.layout.css, gu.mb.contents.css

2.2. 클래스명 정의

  • 스타일 정의시 아이디(#) 사용은 피한다.
  • 클래스명(.)은 소문자로 이루어지며, 대문자를 사용하지 않는다.
  • 단어와 단어의 연결은 Underscore(_)로 한다.
    ex) .list_data, .tbl_data, ... (O)
         .list-data, .tbl-data, ... (Ⅹ)
    (단, 플러그인 커스터마이징 또는 스크립트 기능을 위한 경우에는 대문자, Hyphen(-) 사용을 허용한다.)
  • 동일한 용도의 클래스이나 스타일이 다르게 정의될 경우, 클래스명 뒤에 숫자를 붙여 구분한다.
    (단, 1은 생략해도 무방하다.)
    ex) .list_item, .list_item2, .list_item3, ...
  • common style 중 margin, padding, width, height 정의 클래스명은 기존에 정의된 클래스를 사용한다.
    .mgt10, .pdb0, .w20p 등 ...

2.3. 스타일(속성) 정의

  • Common Style의 경우 margin, padding, width, height 정의 클래스명은 기존에 정의된 내용을 사용한다.
    (필요에 따라 내용 추가)
  • 예약어는 단독으로 정의하지 않고, 반드시 상속을 통해 정의한다.
    * 예약어 .btn, .tit, .txt, .msg, .img, .cont, .list, .tbl, .info, .name, .data, .on, .off, .small, .large, .color ...
  • Common Style 외에 !important 사용을 지양한다. 상속이나 추가 클래스를 통해 그 우선순위를 결정한다.
  • 사이즈의 상대단위, 절대단위는 정의된 디자인에 따라 적절하게 사용한다.
    단, 폰트 사이즈 단위는 rem을 기본으로 한다.
  • 속성의 정의 순서를 준수한다.(권장사항)
    display → position → float → width, height → margin, padding → border → background → font → 기타...
  • 스크립트를 통한 스타일 정의는 가급적 피한다.

3. IMAGES

3.1. 위치 및 정의

  • /images/*/*.*
    • /gu.common/ *.png
      header, footer, button, form 등 공통 요소에 사용되는 이미지
    • /gu.contents/ *.png
      공통 요소를 제외한 모든 이미지

  • ※ 기본적으로 ratio x2 적용, 필요에 따라 x3, x4 적용

    ※ slplit image 사용 권장

  • ② Desktop, Mobile 구분
    /gu.mb.common/, /gu.mb.contents/ *
  • ※ 웹폰트 사용할 경우
    └ /font/ 폴더에 폰트 파일 저장

3.2. 파일명 정의

  • 네이밍의 순서는 '형태-의미-상태' 순
    ex) bg_arr_on.png , ico_check_off.png
  • 동일한 용도의 이미지가 여러개인 경우 마지막 단어에 숫자를 붙여서 다르게 명명한다.
    ex) bg_arr.png, bg_arr02.png
  • 영역을 표시하기 위한 임시이미지나 운영화면에서는 서버에서 불러와서 처리하는 이미지는 @을 앞에 붙여 구분한다.
    ex)@banner_main.gif
